# Динамический массив (Vector)

Этот документ описывает пошаговую реализацию класса Vector - аналога std::vector из стандартной библиотеки C++. Vector представляет собой динамический массив, который может изменять свой размер во время выполнения программы.

## 1. Основная структура класса

Первым шагом создадим базовую структуру класса с конструкторами и деструктором. Конструкторы позволяют создавать векторы разными способами, а деструктор освобождает выделенную память.

```cpp
#include <iostream>
#include <cstddef>

template <typename T>
class Vector {
public:
  Vector();                                    // создает пустой вектор
  Vector(size_t arr_size);                     // создает вектор заданного размера
  Vector(size_t arr_size, const T& elem);      // создает вектор с одинаковыми элементами

  ~Vector();                                   // освобождает память

private:
  T *data;                  // указатель на массив данных
  size_t vectorSize;        // текущее количество элементов
  size_t vectorCapacity;    // максимальное количество элементов без перевыделения памяти
};
```

**Важные моменты реализации:**
- Конструктор должен выделять память эффективно
- Начальная ёмкость должна быть разумной (например, 1 для непустых векторов)
- Деструктор обязательно должен освободить всю выделенную память
- `vectorSize` хранит количество реально используемых элементов
- `vectorCapacity` показывает, сколько элементов может поместиться без расширения массива

### Примеры использования
```cpp
  Vector<int> vec1;           // пустой вектор []
  Vector<int> vec2(5);        // вектор из 5 нулей [0, 0, 0, 0, 0]
  Vector<int> vec3(3, 42);    // вектор из 3 элементов со значением 42: [42, 42, 42]
```

## 2. Семантика копирования и перемещения

Эти функции обеспечивают правильное копирование и эффективное перемещение объектов. Без них компилятор создаст свои версии, которые могут работать неправильно с динамической памятью.

```cpp
  Vector(const Vector<T> &other);              // конструктор копирования
  Vector(Vector<T> &&other);                   // конструктор перемещения

  Vector<T> &operator=(const Vector<T> &other); // оператор присваивания копированием
  Vector<T> &operator=(Vector<T> &&other);      // оператор присваивания перемещением
```

**Что должны делать эти функции:**
- **Конструктор копирования**: создает полную независимую копию другого вектора (глубокое копирование)
- **Конструктор перемещения**: "крадет" данные у временного объекта, не копируя их
- **Операторы присваивания**: присваивают содержимое одного вектора другому, уже существующему

**Правило "Rule of Five"**: если класс управляет ресурсами (как наш Vector с динамической памятью), нужно определить все пять функций: деструктор, конструктор копирования, оператор копирования, конструктор перемещения и оператор перемещения.

### Пример использования
```c++
  Vector<int> vec1(3, 10);    // создаем вектор [10, 10, 10]
  Vector<int> vec2(vec1);     // копируем vec1 в vec2
  Vector<int> vec3 = std::move(vec1);  // перемещаем данные из vec1 в vec3
  
  // vec1 теперь пустой (или в неопределенном, но валидном состоянии)
  // vec2 и vec3 содержат [10, 10, 10]
  
  std::cout << "vec2: ";
  for(size_t i = 0; i < vec2.size(); ++i) {
    std::cout << vec2[i] << " ";
  }
  // Вывод: vec2: 10 10 10
```

## 3. Информация о размере и ёмкости

Эти функции позволяют получить информацию о текущем состоянии вектора без его изменения.

```cpp
  size_t size() const;        // возвращает количество элементов
  size_t capacity() const;    // возвращает ёмкость (размер выделенной памяти)
  bool empty() const;         // проверяет, пустой ли вектор
```

**Объяснение концепций:**
- **Размер (size)** - сколько элементов фактически хранится в векторе
- **Ёмкость (capacity)** - сколько элементов может поместиться без перевыделения памяти
- **Пустота (empty)** - удобная функция для проверки `size() == 0`

Ёмкость всегда больше или равна размеру. Когда размер достигает ёмкости, вектор должен увеличить свою ёмкость.

### Пример использования
```cpp
  Vector<int> vec(5, 100);
  std::cout << "Размер: " << vec.size() << '\n';      // Размер: 5
  std::cout << "Ёмкость: " << vec.capacity() << '\n'; // Ёмкость: 5 (или больше)
  std::cout << "Пустой: " << vec.empty() << '\n';     // Пустой: 0 (false)
```

## 4. Доступ к элементам

Эти операторы позволяют читать и изменять элементы вектора по индексу.

```cpp
  const T &operator[](size_t index) const;  // чтение элемента (const версия)
  T &operator[](size_t index);              // изменение элемента

  const T &at(size_t index) const;          // безопасное чтение с проверкой границ
  T &at(size_t index);                      // безопасное изменение с проверкой границ
```

**Различия между операторами:**
- **`operator[]`** - быстрый доступ без проверки границ (как в обычных массивах)
- **`at()`** - безопасный доступ с проверкой границ, бросает исключение при выходе за пределы

**Зачем нужны const и non-const версии:**
- const версии используются для объектов, объявленных как const
- non-const версии позволяют изменять элементы

### Пример использования
```cpp
  Vector<int> vec(3, 0);      // создаем вектор [0, 0, 0]
  vec[0] = 10;                // быстрое присваивание
  vec[1] = 20;
  vec[2] = 30;
  
  std::cout << "vec[1]: " << vec[1] << '\n';          // vec[1]: 20
  std::cout << "vec.at(2): " << vec.at(2) << '\n';    // vec.at(2): 30
  
  // vec.at(5) бросит исключение std::out_of_range, так как индекс 5 не существует
  // vec[5] может привести к неопределенному поведению (обращение к чужой памяти)
```

## 5. Операции изменения содержимого

Эти функции позволяют динамически изменять содержимое вектора: добавлять и удалять элементы.

```cpp
  void push_back(const T &x);   // добавляет копию элемента в конец
  void push_back(T &&x);        // добавляет элемент в конец с перемещением
  
  void pop_back();              // удаляет последний элемент
  
  void clear();                 // удаляет все элементы (размер становится 0)
```

**Подробности реализации:**
- **`push_back()`** должен автоматически увеличивать ёмкость при необходимости
- Обычная стратегия роста - удваивать ёмкость при нехватке места
- **`pop_back()`** только уменьшает размер, не освобождает память (ёмкость остается прежней)
- **`clear()`** обнуляет размер, но может оставить ёмкость неизменной для эффективности

### Пример базового использования
```cpp
int main() {
  Vector<int> vec;            // пустой вектор
  vec.push_back(1);           // добавляем элементы
  vec.push_back(2);
  vec.push_back(3);
  
  std::cout << "Размер: " << vec.size() << '\n';  // Размер: 3
  
  vec.pop_back();             // удаляем последний элемент
  std::cout << "Размер после pop: " << vec.size() << '\n';  // Размер после pop: 2
  
  vec.clear();                // очищаем вектор
  std::cout << "Размер после clear: " << vec.size() << '\n';  // Размер после clear: 0
}
```

### Пример динамического роста ёмкости
Этот пример показывает, как вектор автоматически увеличивает свою ёмкость при добавлении элементов:

```cpp
  Vector<int> vec;            // начинаем с пустого вектора
  std::cout << "Начальная ёмкость: " << vec.capacity() << '\n';  // 0
  
  vec.push_back(1);           // добавляем первый элемент
  std::cout << "После 1 элемента: " << vec.capacity() << '\n';   // 1
  
  vec.push_back(2);           // ёмкость увеличивается
  std::cout << "После 2 элементов: " << vec.capacity() << '\n';  // 2
  
  vec.push_back(3);           // ёмкость снова увеличивается
  std::cout << "После 3 элементов: " << vec.capacity() << '\n';  // 4 (удвоение)
  
  // Теперь можно добавить еще один элемент без перевыделения памяти
  vec.push_back(4);
  std::cout << "После 4 элементов: " << vec.capacity() << '\n';  // всё ещё 4
```

## Общие принципы реализации

**Управление памятью:**
- Всегда используйте `new[]` и `delete[]` для выделения/освобождения массивов
- Следите за тем, чтобы не было утечек памяти
- При росте ёмкости скопируйте все старые элементы в новый массив

**Эффективность:**
- Амортизированная сложность `push_back()` должна быть O(1)
- Избегайте частых перевыделений памяти
- Используйте перемещение вместо копирования, когда это возможно

**Безопасность:**
- Всегда проверяйте границы в функции `at()`
- Обрабатывайте особые случаи (пустые векторы, self-assignment и т.д.)
- Следите за корректностью работы с исключениями
