# Variadic Templates и Universal References в C++

## Что такое Variadic Templates?
Variadic Templates — это шаблоны, которые могут принимать переменное количество аргументов разных типов.
Они позволяют создавать функции и классы, работающие с произвольным числом параметров типобезопасным способом.

## Что такое Universal References?
Universal References (также называемые forwarding references) — это особый вид ссылок `T&&`, которые могут связываться как с lvalue, так и с rvalue объектами. Они используются вместе с perfect forwarding для эффективной передачи аргументов.

Дополнительная информация:
- Variadic Templates: [cppreference.com](https://en.cppreference.com/w/cpp/language/parameter_pack)
- Universal References: [Scott Meyers - Universal References](https://isocpp.org/blog/2012/11/universal-references-in-c11-scott-meyers)
- Perfect Forwarding: [cppreference.com](https://en.cppreference.com/w/cpp/utility/forward)

## Простой пример Variadic Templates
```cpp
#include <iostream>

template<typename... Args>
void print(Args... args) {
    ((std::cout << args << " "), ...);  // C++17 fold expression
    std::cout << '\n';
}

int main() {
    print(1, 2.5, "hello", 'c');  // 1 2.5 hello c
    return 0;
}
```

## Простой пример Universal References
```cpp
#include <iostream>
#include <utility>

template<typename T>
void process(T&& arg) {
    // Perfect forwarding
    forward_to_another(std::forward<T>(arg));
}
```

## Практические задания

### Variadic Templates

#### 1. Функция для вывода произвольного количества аргументов
Создайте функцию-шаблон `printAll`, которая принимает произвольное количество аргументов любых типов и выводит их через пробел.
Используйте C++17 fold expressions для реализации.

#### 2. Вычисление суммы произвольного количества чисел
Напишите функцию-шаблон `sum`, которая:
- Принимает произвольное количество числовых аргументов
- Возвращает их сумму
- Работает с разными числовыми типами (`int`, `double`, `float`)

Используйте fold expression для суммирования.

#### 3. Поиск максимального элемента
Создайте функцию-шаблон `findMax`, которая:
- Принимает произвольное количество аргументов одного или разных типов
- Возвращает максимальный элемент
- Используйте fold expression с оператором сравнения

#### 4. Подсчёт количества аргументов определённого типа
Напишите функцию-шаблон `countType<T>`, которая:
- Принимает произвольное количество аргументов
- Возвращает количество аргументов типа `T`
- Используйте `std::is_same_v` для проверки типов

#### 5. Создание tuple из аргументов
Создайте функцию-шаблон `makeTuple`, которая:
- Принимает произвольное количество аргументов
- Возвращает `std::tuple`, содержащий все переданные аргументы
- Проверьте работу с разными типами данных

#### 6. Рекурсивная функция обработки аргументов (до C++17)
Реализуйте функцию `processArgs` без использования fold expressions:
- Используйте рекурсивный подход с базовым случаем
- Функция должна выводить каждый аргумент на новой строке
- Покажите разницу между рекурсивным подходом и fold expressions

#### 7. Variadic класс-шаблон для хранения разнотипных данных
Создайте класс-шаблон `VariadicContainer<Types...>`, который:
- Хранит данные разных типов в `std::tuple`
- Имеет конструктор, принимающий произвольное количество аргументов
- Предоставляет метод `get<Index>()` для получения элемента по индексу
- Имеет метод `printAll()` для вывода всех элементов

#### 8. Функция для применения операции ко всем аргументам
Напишите функцию-шаблон `applyToAll`, которая:
- Принимает функцию/лямбду и произвольное количество аргументов
- Применяет функцию к каждому аргументу
- Возвращает результаты в виде tuple

### Universal References и Perfect Forwarding

#### 9. Wrapper функция с perfect forwarding
Создайте функцию-шаблон `callFunction`, которая:
- Принимает указатель на функцию и произвольное количество аргументов
- Использует universal references для аргументов
- Передаёт аргументы в функцию с помощью `std::forward`
- Возвращает результат вызова функции

#### 10. Фабричная функция с perfect forwarding
Напишите функцию-шаблон `makeObject<T>`, которая:
- Принимает произвольное количество аргументов для конструктора
- Создаёт объект типа `T`, передавая аргументы в конструктор
- Использует perfect forwarding для эффективной передачи
- Возвращает созданный объект

#### 11. Логгер с perfect forwarding
Создайте класс `Logger` с методом-шаблоном `log`, который:
- Принимает произвольное количество аргументов с universal references
- Выводит все аргументы в лог с временной меткой
- Использует perfect forwarding для избежания лишних копирований

#### 12. Измерение времени выполнения функций
Создайте функцию-шаблон `measureTime`, которая:
- Принимает функцию и её аргументы с universal references
- Измеряет время выполнения функции
- Использует perfect forwarding для передачи аргументов
- Возвращает как результат функции, так и время выполнения

#### 13. Кэширующий wrapper
Напишите класс-шаблон `CachedFunction`, который:
- Обёртывает функцию и кэширует её результаты
- Использует variadic templates для поддержки функций с разным количеством параметров
- Применяет perfect forwarding для эффективной передачи аргументов
- Использует `std::map` для хранения кэша

#### 14. Событийная система
Создайте простую событийную систему:
- Класс `EventManager` с методом `subscribe<EventType>`
- Метод `emit<EventType>` для генерации событий с произвольными аргументами
- Используйте variadic templates и perfect forwarding
- Обработчики событий должны получать аргументы без лишних копирований

#### 15. Конструктор объектов в векторе
Создайте функцию-шаблон `emplaceInVector`, которая:
- Принимает ссылку на `std::vector<T>` и аргументы для конструктора
- Использует perfect forwarding для передачи аргументов в `emplace_back`
- Работает с любыми типами объектов и их конструкторами

#### 18. Мета-функции для работы с типами
Создайте набор мета-функций:
- `all_same_v<Types...>` — проверяет, что все типы одинаковые

- `last_type_t<Types...>` — получает последний тип из пакета

Используйте эти мета-функции в практических примерах.
